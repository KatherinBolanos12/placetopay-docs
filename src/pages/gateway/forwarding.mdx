### Forwarding

Es una configuración a nivel de comercios, necesaria cuando  un sitio cuenta con más de un medio de pago de la misma marca (Visa y Mastercard), pero con diferentes proveedores y se requiere definir un adquirente principal.

Esta funcionalidad le permite al comercio determinar cuál proveedor y adquirente debe utilizar en cada transacción con tarjeta de crédito corriente, asegurando un flujo consistente entre los diferentes procesadores disponibles.

<Note>Esto aplica solo para transacciones con tarjeta Visa, Mastercard y tipo de crédito pago corriente.</Note>

### ¿Cómo verificar si un medio de pago tiene configurado forwarding?

Cuando se realiza una solicitud al endpoint de información, la propiedad `hasForwarding` siempre está presente en la respuesta. Sin embargo, su valor será `true` únicamente cuando la funcionalidad de forwarding esté configurada correctamente. Esto indica que el sistema puede dirigir las transacciones al proveedor adecuado según las reglas definidas para el medio de pago.

### Comportamiento de créditos en el flujo de forwarding

Cuando se obtiene una respuesta donde la propiedad `hasForwarding` es `true`, es fundamental que los créditos incluidos en esa misma respuesta sean almacenados en una memoria temporal. Estos créditos llegan junto con la primera solicitud de información y no deben ser modificados en las siguientes solicitudes.

En todas las transacciones y solicitudes posteriores a la primera solicitud de información, se deben trabajar y mostrar al usuario final los mismos créditos obtenidos inicialmente. Esto garantiza consistencia y precisión en el flujo, especialmente en la solicitud de procesamiento u otras interacciones relacionadas.

De esta forma, el forwarding opera correctamente, asegurando que las reglas configuradas se respeten y se eviten inconsistencias en los datos enviados.

### Ejemplo de solicitud para forwarding

Cuando la propiedad **hasForwarding** se establece en `true` en la petición **solicitud de Información**, es necesario incluir el atributo **credit** en el objeto **instrument** de las siguientes solicitudes:
- **Solicitud de Información**
- **3DS Lookup**
- **Generación de OTP**
```json
    // ...
    "instrument": {
        "card": {
            "number": "4111111111111111",
            "expiration": "12/31",
            "cvv": "303"
        },
        "credit": {
            "code": "1",
            "type": "00",
            "groupCode": "C",
            "installments": 1
        }
    }
    // ...
```
Para más información sobre **credit**, revisar la documentación de [Credit](/gateway/additional-data#additional-data-credit).
