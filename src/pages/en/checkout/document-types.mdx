# Document types

Placetopay API supports different document types depending on the country.

To define or use the `payer`, `buyer`, `shipping` and other attributes, you must append the `document` and `documentType` attributes. These data are validated by the API to ensure the format of the information.

Next we indicate the possible values of this field:

### Global

Code| Document type | validation rule
---------|----------|----------
 `PPN` | Passport | `'/^[a-zA-Z0-9_]{4,16}$/'`
 `TAX` | TAX | `'/^[a-zA-Z0-9_]{4,16}$/'`
 `LIC` | Labeler Identification Code | `'/^[a-zA-Z0-9_]{4,16}$/'`

### Puerto Rico (PR)

For Puerto Rico it is not required to send the `documentType` and `document` fields.

 Code| Document type| validation rule
---------|----------|------------
 `EIN` | Employer Identification Number|`'/^[1-9]\d?-\d{7}$/'`


### Colombia (CO)

Code| Document type | validation rule
---------|---------- |----------
 `CC` | Cédula de ciudadanía  | `'/^[1-9][0-9]{3,9}$/'`
 `CE` | Cédula de extranjería | `'/^([a-zA-Z]{1,5})?[1-9][0-9]{3,7}$/'`
 `TI` | Tarjeta de identidad| `'/^[1-9][0-9]{4,11}$/'`
 `NIT` | Número de Identificación Tributaria|` '/^[1-9]\d{6,9}$/'`
 `RUT` | Registro único tributario| `'/^[1-9]\d{6,9}$/'`

### Ecuador (EC)

Code| Document type | validation rule
---------|----------|---------
 `CI` | Cédula de identidad|`'/^\d{10}$/'`
 `RUC` | Registro Único de Contribuyentes|`'/^\d{13}$/'`


### Costa Rica (CR)

Code| Document type | validation rule
---------|----------|------------

### Chile (CL)

Code| Document type | validation rule
---------|----------|------------
 `CLRUT` | Cédula personal física |`'/^(\d{1,2}(?:\.?\d{1,3}){2}-[\dkK])$/'`


### Panamá (PA)

Code| Document type| validation rule

### Brasil (BR)
