# Historial de cambios - Changelog

Este archivo contiene las mejoras y actualizaciones que se hagan en Checkout API.

## Sin desplegar {{ id: 'undeployed' }}

...

## `2.17.33` 2023-10-20

### Cambió:

- Mejora en la validación de las propiedades `instrument` y `payment` para la solicitud del endpoint `api/collect`.
- Mejora en la validación de las propiedades `payment` y `subscription` para la petición de crear una sesión `/api/session`.

## `2.17.32` 2023-10-11

### Nuevo:

- Se agrega modificadores de pago al `/api/collect`. Ahora, cuando se recollecta una suscripción y el sitio admite el cobro de tarifas de transacción, este costo se procesará mediante un pago de tipo dispersión.

## `2.17.30` 2023-09-26

### Nuevo:

- Se agrega control de idempotencia en el endpoint de collect `/api/collect`.

## `2.17.29` 2023-09-26

### Nuevo:

- Se agrega soporte para la configuración del sitio con tarifa de transacción.  Ahora podrán mostrar el impuesto y el costo de una transacción, el cobro de este costo se realizará mediante un pago de tipo dispersión `/api/session` endpoint. 

## `2.17.24` 2023-09-12

### Cambió:

- Se agrega soporte para los caracteres `<,>,=,:,#,%,&,@,=` propiedad `*.description` y `*.reference` en la petición de crear una sesión `/api/session`.

## `2.17.21` 2023-09-01

### Cambió:

- Se agrega soporte para corchetes `[]` para la propiedad `*.description` en la petición de crear una sesión `/api/session`.

## `2.17.20` 2023-08-24

### Nuevo:

- Se agrega soporte de idioma para el endpoint Reversar pago `/api/reverse`.

## `2.17.18` 2023-08-14

### Cambió:

- Permite en la petición para crear una sesión `/api/session` que la propiedad `payment.dispersion.[].amount.total` se puedan establecer en zero.

## `2.17.16` 2023-08-14

### Nuevo

- Mejora en validación de la propiedad `payment.amount.total` en la petición para crear una sesión `/api/session`. Ahora es requerido, debe ser numérico y mayor a cero.

### Cambió:

- Mejora en el manejo de la propiedad `fields`, `payment.fields` y `subscription.fields` mostradas en el resumen de una sesión, endpoint `/api/session`.  Ver más en  [Campos Adicionales](/checkout/additional-fields).  

## `2.17.1` 2023-06-01

### Nuevo

- Permite invalidar un token `InstrumentInvalidateRequest` usando el `subtoken`.

## `2.17.0` 2023-05-25

### Nuevo

- Se agrega el endpoint **Instrument Invalidate** `/api/instrument/invalidate` que sirve para invalidar tokens ya existentes del sitio.
