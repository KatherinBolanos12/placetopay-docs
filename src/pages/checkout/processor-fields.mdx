# Datos de procesamiento

La propiedad `processorFields` proporciona información adicional sobre el procesamiento de una sesión finalizada. Cada objeto se guarda en `payment.processorFields` con el tipo [NameValuePair](additional-fields#name-value-pair).

```json
{
    "requestId": 7,
    "status": { /* ... */ },
    "request": { /* ... */ },
    "payment": [
        {
            "processorFields": [
                {
                    "value": "00990101",
                    "keyword": "terminalNumber",
                    "displayOn": "none"
                },
                {
                    "value": "529259",
                    "keyword": "bin",
                    "displayOn": "none"
                },
                {
                    "value": "0745",
                    "keyword": "lastDigits",
                    "displayOn": "none"
                },
                {
                    "value": "b466b12c2a248122c9448783c77435ed",
                    "keyword": "id",
                    "displayOn": "none"
                },
                {
                    "value": "000",
                    "keyword": "b24",
                    "displayOn": "none"
                },
                /// ...
            ],
            // ...
        }
    ],
}
```

Adicional a estos datos, hay datos que varian según el método de pago, como:

## Descripción del objeto `Credit` {{ id: "description-credit" }}

A continuación se describen las posibles las propiedades del objeto `credit` y los valores que pueden encontrarse en ellas:

### Propiedad `type`

La propiedad `type` define el tipo del crédito y pueden encontrarse los siguientes valores:

| Tipo                                              | Descripción                  |
|---------------------------------------------------|------------------------------|
| `00`                                              | Corriente                    |
| `01`                                              | Diferido Corriente           |
| `02`                                              | Diferido propio              |
| `03`                                              | Plan pagos especiales        |
| `11`                                              | Plan pagos normal            |
| `12`                                              | Diferido corriente con interés|
| `21`                                              | Diferido plus cuotas         |
| `22`                                              | Diferido plus pago total     |

### Propiedad `groupCode`

La propiedad `groupCode` define el código del grupo y es usado para definir el campo de la descripción y pueden encontrarse los siguientes valores:

| groupCode                                        | Descripción                  |
|---------------------------------------------------|------------------------------|
| `C`                                              | Corriente                    |
| `D`                                              | Diferido Corriente           |
| `P`                                              | Diferido propio              |
| `X`                                              | Plan pagos especial          |
| `N`                                              | Plan pagos normal            |
| `I`                                              | Diferido corriente con interés|
| `M`                                              | Diferido plus cuotas         |
| `T`                                              | Diferido plus pago total     |
| `L`                                              | Diferido preferente          |

### Propiedad `code`

La propiedad `code` define el identificador del matriz, y sus valores posibles son 0 ó 1.

### Propiedad `installments`

La propiedad `installments` define la cantidad de cuotas permitidas en el crédito.
